## Getting Started

### Calculator Layout
The calculator features a Casio-style design with:
- **Display Screen**: Shows current expression and results
- **Function Buttons**: Blue buttons for scientific functions
- **Operator Buttons**: Orange buttons for mathematical operations
- **Number Pad**: White buttons for digits and decimal point
- **Memory Buttons**: Standard memory operations (MC, MR, M+, M-)
- **Special Functions**: Advanced mathematical and programming functions

### Basic Controls
- **AC**: Clear all (display, expression, memory)
- **C**: Clear current entry (display and expression)
- **SHIFT**: Toggle scientific mode (for future expansion)

The calculator follows standard mathematical order (PEMDAS/BODMAS):
1. Parentheses/Brackets
2. Exponents/Roots
3. Multiplication/Division
4. Addition/Subtraction

**Example**: `3 + 4 × 2 = 11` (not 14)

## Memory Functions

### Memory Operations
- **MC**: Memory Clear - Clears all memory
- **MR**: Memory Recall - Displays stored value
- **M+**: Memory Plus - Adds current value to memory
- **M-**: Memory Minus - Subtracts current value from memory

### Memory Usage Example
```
1. Enter 50 and press M+ (Memory = 50)
2. Enter 25 and press M+ (Memory = 75)
3. Enter 10 and press M- (Memory = 65)
4. Press MR to recall 65
5. Press MC to clear memory
```

## Troubleshooting

### Common Issues

#### Input Appending to Results
**Problem**: After getting a result, new numbers append to it
**Solution**: This is normal behavior. Start a new calculation by:
- Pressing any operator (+, -, ×, ÷) to continue calculation
- Pressing C to clear and start fresh
- Pressing AC to clear everything

#### Display Overflow
**Problem**: Long expressions don't fit in display
**Solution**: The calculator automatically truncates with "..." for readability

#### Error Messages
**Problem**: Display shows "Error"
**Solution**: 
- Check for invalid operations (division by zero)
- Verify function syntax
- Press C or AC to clear and retry

#### Unexpected Results
**Problem**: Trigonometric functions give wrong answers
**Solution**: Check angle mode (DEG vs RAD vs GRAD)

#### Memory Issues
**Problem**: Memory functions not working
**Solution**: 
- Ensure you have values stored (M indicator appears)
- Use MR to verify memory content
- Press MC to clear corrupted memory

### Calculation Tips

1. **Use parentheses** for complex expressions to ensure correct order
2. **Check angle mode** before trigonometric calculations
3. **Clear memory** before important calculations
4. **Use DATA+** carefully for statistics - wrong entries affect all results
5. **Press =** after each major calculation to verify intermediate results

- Number keys for direct input
